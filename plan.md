# 项目名称：TravelNet — 智能周边城市探索与旅行决策系统

## 角色
你是一名全栈前端开发专家，擅长数据可视化、地理信息处理和现代Web UI设计。请帮我生成一个**完整、可直接运行**的HTML/CSS/JS单页应用。

## 核心目标
创建一个网页，用户打开后：
1. 获取当前地理位置（或允许手动输入坐标）。
2. 以该位置为中心，筛选出周围500公里范围内的**所有小城市**（城市列表内置，需包含名称、经纬度、简短简介）。
3. 将筛选出的城市以**高科技感的网络图**形式展示：中心点为“当前位置”，每个城市是一个节点，用射线或曲线连接中心与节点，节点的半径/颜色/标签可反映距离或类型。布局需真实反映方位角与距离（例如：正北向上，距离映射为半径，角度按实际方位）。
4. 支持以下交互功能：
   - **随机选择一个城市**：从未被“确定排除”的城市中随机抽取，弹窗或浮动卡片展示城市名称、距离、简介，并附带【确定】和【取消】按钮。
     - 点击【确定】：该城市标记为“已旅游/已确定”，从后续随机池中移除，并自动添加到“我的旅游记录”中（允许稍后补充感想照片）。
     - 点击【取消】：不做任何标记，城市仍留在随机池中，可再次被随机到。
   - **搜索框**：输入城市名称，实时过滤并在网络图中高亮匹配节点（例如放大节点、改变颜色或闪烁）。
   - **我的旅游记录**：侧边栏或独立面板，展示所有已确定/手动添加的城市。每个记录项可：
     - 添加/编辑文字感想（textarea）
     - 上传照片（多张，预览缩略图，存储为base64或图片URL）
     - 删除该记录（同时从随机池中恢复？用户可自定义：删除时询问是否重新加入随机池，建议默认“仅移除记录，不恢复随机池”，但提供“恢复候选”按钮）。
   - **手动添加已去城市**：如果用户去过某个不在随机列表中的城市（或未通过随机确认），可手动添加城市（搜索城市名或从完整城市库中选择），并同样支持感想照片。
5. **数据持久化**：所有数据（已确认排除的城市ID、旅游记录及感想照片）保存在 `localStorage` 中，页面刷新后恢复状态。

## 视觉与体验要求
- 风格：**科技感、炫酷、深色背景**，网络图带有发光线条、粒子效果或光晕，节点悬停显示城市信息。
- 响应式布局：桌面端为主，但移动端基本可用。
- 交互流畅：随机选择时伴有动画或音效（可选），确认/取消时有明确反馈。
- 网络图应支持缩放、拖拽（如果使用SVG/Canvas），同时展示距离标尺或图例。

## 技术栈（推荐）
- 纯前端：HTML5, CSS3, ES6+
- 可视化库：**D3.js** 或 **Three.js**（建议D3.js实现2.5D效果，降低复杂度；如果你擅长Three.js也可以用3D）
- 地理计算：Haversine公式计算距离、方位角
- 存储：localStorage
- 图标/UI：FontAwesome 或 无外部依赖（可自定义）
