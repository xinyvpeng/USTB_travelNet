# 发现与决策

## 需求
- 基于固定位置（北京科技大学：北纬39.99048°，东经116.36087°）展示500公里内县级城市
- 网络图可视化：中心点为当前位置，节点为城市，距离映射为半径，方位角准确
- 交互功能：随机选择城市、搜索过滤、旅游记录管理（感想+照片）、手动添加城市
- 数据持久化：用户数据本地存储，页面刷新后恢复
- 科技感深色主题，响应式设计
- 部署到GitHub Pages（静态托管）
- **用户认证系统**：匿名用户只读访问，认证用户可编辑，保护内容不被随意修改

## 研究发现
- Vite构建工具在Windows环境下需要正确的native模块，rollup的Windows native模块有时会出错，删除node_modules和package-lock.json重新安装可解决
- D3.js版本7.9.0与ES模块导入兼容良好
- localforage简化了IndexedDB操作，适合存储用户照片和记录
- 北京周边500公里内包含约60-80个县级城市，覆盖北京、天津、河北、山西、内蒙古、辽宁、山东等地区
- 深色科技主题使用CSS变量和渐变效果实现良好视觉效果
- 网络图布局需要将经纬度转换为极坐标（距离映射为半径，方位角映射为角度） 500公里范围大约覆盖北京、天津、河北、山西、内蒙古、辽宁、山东等部分地区
- 县级城市数量估计：80-100个（需要具体数据收集）
- Haversine公式适用于计算球面距离
- D3.js适合网络图可视化，有丰富示例
- IndexedDB可提供更大存储空间（相比localStorage的5MB限制）
- 照片存储需要考虑压缩和大小限制

## 技术决策
| 决策 | 理由 |
|------|------|
| 分阶段开发 | 降低复杂度，逐步验证，便于调整方向 |
| 数据源：内置静态JSON | 避免外部API依赖，简化部署，离线可用 |
| 可视化库：D3.js | 成熟的可视化库，社区支持好，2.5D效果足够 |
| 存储方案：IndexedDB + localForage | 支持大容量存储，简化API使用 |
| 构建工具：Vite | 现代构建工具，开发体验优秀，生产优化好 |
| 部署平台：GitHub Pages | 免费静态托管，集成GitHub工作流 |
| 无UI框架（原生JS/CSS） | 轻量，无依赖，适合小型项目 |
| 照片压缩：200KB/张上限 | 平衡质量与存储空间 |
| 认证系统：前端密码验证 | 简单实现，适合个人项目，无需后端服务 |

## 遇到的问题
| 问题 | 解决方案 |
|------|---------|
| 照片存储空间限制 | 使用IndexedDB替代localStorage，添加压缩功能 |
| 网络图性能 | 限制显示节点数量，实现虚拟化或分页 |
| 城市数据收集 | 使用公开数据集（如GeoNames）或手动整理 |
| 方位角计算 | 使用数学公式计算两点之间的方位角 |
| 跨浏览器兼容性 | 测试主流浏览器，使用polyfill必要时 |
| rollup native模块错误 | 删除node_modules和package-lock.json重新安装，确保正确安装Windows原生模块 |
| 构建失败 | 检查node版本和架构匹配，使用npm rebuild必要时 |
