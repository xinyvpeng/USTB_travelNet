# 任务计划：TravelNet — 智能周边城市探索与旅行决策系统

## 目标
创建一个基于北京科技大学位置（北纬39.99048°，东经116.36087°）的单页应用，展示500公里内县级城市网络图，支持随机选择、搜索、旅游记录管理，并部署到GitHub Pages。

## 关键问题
1. 城市数据来源：需要确保数据准确性（经纬度、人口）
2. 照片存储方案：IndexedDB存储上限与压缩策略
3. 网络图性能：节点数量优化与渲染效率
4. 用户体验：复杂交互的直观性

## 已做决策
| 决策 | 理由 |
|------|------|
| 分阶段开发 | 降低风险，逐步验证，便于调整 |
| 数据源：内置静态JSON | 无需外部API依赖，部署简单 |
| 照片存储：IndexedDB | 突破localStorage 5MB限制，支持更大存储 |
| 可视化：D3.js | 2.5D效果足够，复杂度可控，社区支持好 |
| 构建工具：Vite | 开发体验好，生产优化，适合现代前端 |
| 部署：GitHub Pages | 静态托管，免费，无需服务器 |
| UI框架：原生JS+CSS | 轻量，无依赖，学习成本低 |
